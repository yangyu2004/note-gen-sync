# 网络编程知识整理

## 一、网络通信基本概述

### （一）基础概念

* **通信**：实现 A 到 B 的信息传递 。
* **网络**：由结点（终端）和连线（有线 / 无线）构成，达成资源共享 。
* **通信协议**：是通信双方的规范与标准，类似沟通语言，用于传输数据，如 TCP/IP 协议（传输控制协议 / 因特网互联协议 ），HTTP 是上层协议应用，还有 HTTPS 。

### （二）计算机网络分类

包含局域网、城域网、广域网、互联网 ，探讨 “具有防火墙的互联网是否还叫互联网” 这类概念性问题 。

### （三）TCP 与 UDP 协议

* **TCP**：传输控制协议，面向连接，安全可靠但效率稍低 。
* **UDP**：用户数据报协议，面向无连接，存在丢包情况 。

## 二、网络模型

### （一）TCP/IP 四层模型（事实模型 ）

包含网络接口层、网际层、传输层、应用层 。

### （二）五层模型（教学模型 ）

涵盖物理层、数据链路层、网络层、传输层、应用层 。

### （三）OSI 七层模型（理论模型 ）

有物理层、数据链路层、网络层、传输层、会话层、表示层、应用层 。

## 三、TCP 保证数据安全手段（三次握手、四次挥手 ）

### （一）三次握手（建立连接 ）

1. **准备阶段**：为建立可靠信道做准备 。
2. **第一次握手**：Client 发送带 SYN 标志数据包给 Server，告知发送接收正常 。
3. **第二次握手**：Server 回复带 SYN/ACK 标志数据包给 Client，客户端确认自身接收及对方发送正常 。
4. **第三次握手**：Client 发送带 ACK 表示的数据包给 Server，服务器确认自身发送及对方接收正常 。

### （二）四次挥手（断开连接 ）

1. **第一次挥手**：客户端发 FIN，关闭客户端到服务器数据传送 。
2. **第二次挥手**：服务器收 FIN，回 ACK 。
3. **第三次挥手**：服务器关闭与客户端连接，发 FIN 给客户端（全双工连接变为半关闭状态 ） 。
4. **第四次挥手**：发回 ACK 报文确认 。

## 四、浏览器输入 URL 后流程

### （一）浏览器处理

1. **URL 解析**：对输入的 URL 进行解析 。
2. **检查缓存**：浏览器先查自身缓存，缓存未命中则生成请求报文准备发送 。

### （二）寻址

通过 DNS 解析，把域名转成 IP 地址，涉及浏览器缓存、系统缓存、路由器 / ISP 缓存、根域名服务器等缓存环节 。

### （三）建立连接

包含 TCP 三次握手、TLS 握手（HTTPS 场景 ） 。

### （四）请求与响应

1. **发送请求**：浏览器向服务器发送请求 。
2. **服务器处理**：服务器接收并处理请求 。
3. **返回响应**：服务器把处理结果返回给浏览器 。

### （五）渲染与展示

浏览器接收响应后进行页面渲染展示 。

## 五、HTTP 相关

### （一）HTTP1.0 与 HTTP1.1 区别

* **连接方式**：HTTP1.0 默认短连接，每次请求需重新建立连接；HTTP1.1 后默认长连接，开启 Connection: keep - alive ，减少 TCP 四次握手开销 。
* **错误状态响应**：HTTP1.1 引入 24 个新错误状态码 。
* **缓存处理**：HTTP1.1 有更多缓存控制策略 。
* **带宽优化及网络连接**：HTTP1.0 不支持断点续传，HTTP1.1 引入，避免浪费带宽 。

### （二）常见 HTTP 请求状态码

* **200**：成功 。
* **301**：资源永久转移到新地址 。
* **302**：资源临时在别处 。
* **404**：未找到资源 。
* **500**：服务器内部错误 。
* 还有 2xx（成功类 ）、3xx（重定向类 ）、4xx（服务器错误 ）、5xx（服务器错误 ）等分类 。

### （三）URI 和 URL 区别

* **URI**：统一资源标志符，唯一标识资源，不管表示方式，类似身份证号 。
* **URL**：统一资源定位符，不仅标识资源，还指明访问资源的方法和位置，类似家庭住址 ，是 URI 的一种 。

### （四）HTTP 和 HTTPS 区别

* **端口及起始**：HTTP 的 URL 以 http:// 起始，默认端口 80；HTTPS 的 URL 以 https:// 起始，默认端口 443 。
* **安全性和资源消耗**：HTTP 运行在 TCP/IP 协议上，传输内容明文；HTTPS 运行在 SSL/TLS 之上，传输内容加密（对称加密结合非对称加密，用服务器证书验证 ），但 HTTPS 消耗更多服务器资源 。

## 六、IO 相关（BIO、NIO ）

### （一）BIO（Blocking I/O，同步阻塞 IO ）

在 JDK1.4 前，网络通信主流是 BIO 。服务器启动 ServerSocket，客户端用 Socket 通信。默认服务器为每个请求建线程等待，客户端发请求后等服务器线程响应，无则一直等或拒，属于阻塞式 IO 。

### （二）NIO（New I/O，Non - blocking I/O，非阻塞式 IO ）

* **与 BIO 对比**：目的相同，实现方式不同。BIO 以流处理数据，NIO 以块（Buffer ）处理，IO 效率更高 ，NIO 是异步的，可构建非阻塞网络 。
* **核心部分**：
  * **Channel（通道 ）**：提供从文件、网络读数据的渠道，读写数据需经 Buffer，有 FileChannel（文件数据读写 ）、DatagramChannel（UDP 数据读写 ）、ServerSocketChannel（TCP 数据读写 ）、SocketChannel（TCP 数据读写 ）等 。
  * **Buffer（缓冲区 ）**：是容器、特殊数组，能跟踪记录状态变化，有数组 + capacity（容量 ）+position（位置 ）+limit（界限 ）+mark（标记 ） ，常用子类有 ByteBuffer、ShortBuffer 等，常用 ByteBuffer（字节缓冲 ） 。
  * **Selector（选择器 ）**：Selector 选择器监听多个通道事件，单个线程可监听多个客户端通道 。

### （三）NIO 代码示例（以文件读写为例 ）

#### 1. 写数据到本地文件

```java
package com.ncse.yxyx.test;
import java.io.FileOutputStream;
import java.nio.ByteBuffer;
import java.nio.channels.FileChannel;
public class TestNIO {
    public void test() throws Exception {
        //1. 创建输出流
        FileOutputStream fos = new FileOutputStream("basic.txt"); 
        //2. 从流中获取一个通道
        FileChannel fc = fos.getChannel(); 
        //3. 提供一个缓冲区
        ByteBuffer buffer = ByteBuffer.allocate(1024); 
        //4. 往缓冲区存入数据
        String str = "Hello World"; 
        buffer.put(str.getBytes()); 
        //5. 翻转缓存区
        buffer.flip(); 
        //6. 把缓冲区数据写入通道中
        fc.write(buffer); 
        //7. 关闭流
        fos.close(); 
    }
}
```

#### 2. 从本地文件读数据

```java
package com.ncse.yxyx.test;
import java.io.File;
import java.io.FileInputStream;
import java.nio.ByteBuffer;
import java.nio.channels.FileChannel;
public class TestNIO2 {
    public void test2() throws Exception {
        File file = new File("basic.txt"); 
        //1. 创建输入流
        FileInputStream fis = new FileInputStream(file); 
        //2. 得到一个通道
        FileChannel fc = fis.getChannel(); 
        //3. 准备一个缓冲区
        ByteBuffer buffer = ByteBuffer.allocate((int) file.length()); 
        //4. 从通道中读取数据并存到缓冲区中
        fc.read(buffer); 
        System.out.println(new String(buffer.array())); 
        //5. 关闭流
        fis.close(); 
    }
}
```

#### 3. 复制文件（对比 BIO ）

```java
package com.ncse.yxyx.test;
import java.io.FileInputStream;
import java.io.FileOutputStream;
public class CopyBIO {
    public void copy() throws Exception {
        FileInputStream fis = new FileInputStream("basic.txt"); 
        FileOutputStream fos = new FileOutputStream("basic2.txt"); 
        byte[] b = new byte[1024]; 
        while (true) { 
            int res = fis.read(b); 
            if (res == -1) { 
                break; 
            }
            fos.write(b, 0, res); 
        }
        fis.close(); 
        fos.close(); 
    }
}
```

#### 4. NIO 中 Buffer 操作要点

NIO 通道从流获取，双向（可读可写 ）。往通道写数据前，需用 put 存数据到 ByteBuffer，再用 write 写。write 前用 flip 翻转缓冲区，把内部数据位置重置，让数据写入通道 。flip 作用是翻转缓冲区，处理指针（pos 写数据到 lim，读时需翻转让 pos = 0，lim = 原 pos ） 。
